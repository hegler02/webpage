교수자가 마주한 새로운 질문

학생 글을 읽을 때 이제 세 가지를 동시에 확인해야 한다

① 진짜 쓴 것인가
학생이 직접 작성했는가, AI가 생성한 것인가
② 이해하고 있는가
내용을 실제로 알고 있는가, 아니면 글만 있는가
③ 근거가 실제인가
출처와 통계가 존재하는 것인가, 만들어진 것인가

잘 쓴 글일수록 더 정교한 확인이 필요하다

기존 대응의 한계

AI 금지 · 표절 검사 · 문체 판단만으로는 부족하다

AI 글 ≠ 표절
기존 문서와 겹치지 않아도 AI가 쓴 글일 수 있다
문체 ≠ 이해도
완성도 높은 문체가 이해를 보장하지 않는다
사후 감점 ≠ 개선
처벌은 다음 과제를 개선시키지 않는다

필요한 것은 단속이 아니라 구조화된 지도와 검증이다

관점 전환

글이 아니라 사고 과정을 본다

과제는 완성된 글 한 편이 아니라 학생의 사고 구조물이다. 확인해야 할 것:

무엇을 주장하는가
어떻게 분석했는가
어디까지 다루는가
얼마나 깊이 들어갔는가
어떤 순서로 전개했는가
무엇을 근거로 삼았는가

6가지 평가 기준을 교수자의 언어로

축교수자의 실제 피드백배점

이 표가 출제 · 지도 · 채점의 공통 언어다

WHAT — 20점

그래서 네 말이 뭐냐

약한 WHAT
"SNS는 청소년에게 영향을 준다"
→ 주장이 없다. 모두가 아는 사실이다.
강한 WHAT
"SNS 사용 시간이 청소년 자존감에 미치는 영향을 분석하고, 비교 심리 노출을 원인으로 사용 기준 필요성을 검토한다"
→ 무엇을, 어떻게, 왜

확인: 이 글은 무엇에 답하는가 — 첫 문단에서 알 수 있는가

HOW — 20점

어떤 방식으로 볼 것인가

HOW = 주장을 다루는 방법론

비교할 것인가
사례를 분석할 것인가
통계를 볼 것인가
문헌을 검토할 것인가
인터뷰·관찰을 활용할 것인가
HOW 없는 글
WHAT이 있어도 방법론이 없으면 글은 선언으로 끝난다
WHERE — 10점

어디까지 다룰 것인가

WHERE = 과제의 경계. 범위를 모르면 아이디어는 늘어나기만 한다.

분석 대상은 누구인가
기간은 어디까지인가
어떤 사례를 포함하는가
무엇은 이번 과제에서 제외하는가
좋은 WHERE
무엇을 다루는지 뿐만 아니라
무엇을 다루지 않는지도 안다
DEPTH — 20점

얼마나 깊이 들어갔는가

선언 → 원인 → 메커니즘 → 반론 → 실행수준. 어디까지 내려갔는가.

"좋다 / 나쁘다" → 깊이 0
"왜 나쁜가" → 원인 제시
"어떤 구조로 작동하는가" → 메커니즘
"반론과 한계를 다루고 실행 수준까지" → 완전한 DEPTH
TIME — 15점

순서가 논리를 만드는가

TIME = 글의 진행 순서와 인과 흐름

앞 문단이 뒤 문단을 낳는가
서론의 질문이 결론에서 답을 얻는가
분석 순서가 자연스러운가
중간에 논리적 점프가 없는가
목차가 있는 글
→ 흐름이 있는 글
목차는 형식, 흐름은 논리다
EVIDENCE — 15점

근거가 어디 있는가

주장은 증거 위에 서야 한다

데이터 · 통계
문헌 · 논문
사례 · 인터뷰 · 관찰
출처 링크 (실제 접근 가능)
"그럴 것 같다"
"일반적으로 알려져 있다"
→ 근거가 아니다

나쁜 피드백 vs 구조화된 피드백

학생은 "글이 약하다"는 말로는 고칠 수 없다. 축을 지정하면 고친다.

나쁜 피드백
"글이 산만하다"
"더 깊이 써라"
"근거가 부족하다"
구조화된 피드백
WHERE가 없다 — 분석 대상을 좁혀라
DEPTH가 약하다 — 반론과 한계를 포함하라
EVIDENCE가 약하다 — 3문단 주장에 문헌을 붙여라
출제 단계에서 6축 활용

채점 기준을 나중에 붙이지 않는다
출제 단계에서 내장한다

과제 요구사항 예시
WHAT: 핵심 주장을 한 문장으로
HOW: 분석 기준 2개 이상
WHERE: 대상·제외 항목 명시
DEPTH: 반론과 한계 포함
TIME: 문제제기→분석→결론 흐름
EVIDENCE: 신뢰 근거 3개 이상
결과
학생은 출제 시점부터 무엇이 평가되는지 안다
교수자는 이 기준 그대로 채점한다
지도 단계에서 6가지 기준 활용

초안 피드백 = 다시 써주는 것이 아니라
어디를 고칠지 알려주는 것이다

학생 초안 입력
→
기준 분석
→
약점 기준 확인
→
개선 우선순위 제시
→
학생이 직접 수정
AI가 학생 글을 대신 쓰면 안 된다
교수자는 AI 분석을 참고해 지도 방향을 결정한다
평가 단계에서 6축 활용

요구한 것을 평가하고
평가할 것을 미리 요구한다

축핵심 질문배점
WHAT핵심 주장이 한 문장으로 명확한가20
HOW방법론이 WHAT을 실현하는가20
WHERE범위와 경계가 명확한가10
DEPTH아이디어가 실행 수준까지 내려갔는가20
TIME논리적 흐름과 순서가 있는가15
EVIDENCE주장에 측정 가능한 근거가 있는가15
PART 3 — 검증과 평가 하네스

왜 AI 평가만으로는
부족한가

AI가 쓴 글을 AI가 평가할 때 생기는 구조적 문제

3

AI가 평가할 때 생기는 위험

완성도 ≠ 이해도
그럴듯한 글을 사고 깊이로 착각할 수 있다
출처 실존 미확인
논문·통계가 실제로 존재하는지 확인하지 못할 수 있다
이해 여부 불명
학생이 내용을 실제로 알고 있는지 알 수 없다
단일 관점 편향
하나의 AI 판단은 하나의 관점에 갇힌다

그래서 채점 전에 검증 게이트가 필요하고, 평가는 5명의 다관점이 필요하다

검증 게이트 시스템

3개의 게이트가
AI 판정을 교수자 판단으로 연결한다

GATE 1
AI 작성 가능성 탐지
구조·문체 분석
→ 의심 높음 시 Gate 2
GATE 2
구술 검증
5개 질문 직접 확인
→ 3/5 이상 통과
GATE 3
출처·링크 검증
URL·논문·통계 실존 확인
→ 브라우징 ON 필수

게이트는 처벌 장치가 아니다 — 교수자가 확인해야 할 지점을 표시하는 장치다

GATE 1AI 작성 가능성 탐지

확정 판정이 아니라 구술 검증 트리거다

탐지 신호 예시

개인 경험 부재
문체 균일성
구조 과잉 완성도
수업 맥락 부재
근거의 일반성
낮음 / 보통
→ 채점 진행
높음 / 매우 높음
→ Gate 2 구술 검증

핵심: "AI가 썼다"가 아니라 "확인이 필요하다"이다. 최종 판단은 교수자가 한다.

GATE 2구술 검증

학생이 실제로 이해했는지는 말로 확인한다

질문 5가지 유형

Q1 출처 확인 — 어디서 찾았는가
Q2 과정 확인 — 어떻게 작성했는가
Q3 확장 — 한 발 더 나아가면?
Q4 반론 — 이 주장에 반대하면?
Q5 개인화 — 본인 경험과 연결은?
3/5 이상 충분
→ 본인 작성 인정, 채점 진행
2/5 이하 충분
→ 재제출 또는 추가 지도
GATE 3출처·링크 검증

AI 시대의 핵심은 글의 유창함이 아니라
근거의 실존성이다

확인할 것

URL이 실제로 열리는가
논문이 실제로 존재하는가
통계 수치가 원 출처에 있는가
인용 방향이 주장과 같은가
허위 기관·미래 날짜는 없는가
논문 지도 원칙
출처 검증은 선택이 아니라 필수다
브라우징 ON 상태에서만 "검증 완료"다

브라우징 OFF 상태에서는
"출처 검증 완료"라고 말하면 안 된다

상태가능한 것불가능한 것
브라우징 OFF 구조 패턴 분석, 내부 논리 검토 URL·논문·통계 실제 확인
브라우징 ON URL 접속, 논문 검색, 통계 대조 API 기반 완전 자동화
OFF = "구조 분석 완료 (미검증)"
vs
ON = "출처 검증 완료"
